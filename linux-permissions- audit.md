# Linux Permissions Audit

## Overview
This project simulates a real-world cybersecurity task where I reviewed and adjusted file permissions on a Linux system.

## Objective
Ensure that sensitive files have the correct access levels to prevent unauthorized access.

## Tools Used
- Linux
- Command Line (chmod, ls -l)

## Steps Taken
1. Reviewed file permissions using ls -l
2. Identified files with incorrect permissions
3. Used chmod to update access levels

## Findings
Some files had overly permissive access, which could allow unauthorized users to read or modify sensitive data.

## Conclusion
Permissions were corrected to follow the principle of least privilege, improving system security.
